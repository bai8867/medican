# 体质判定算法灰度上线方案

## 目标
- 将现有 `legacy-v1` 九题最高分规则平滑升级到 `v2-research-hybrid`。
- 保留用户手动确认，避免一次性替换造成推荐偏移。

## 分阶段发布

### Phase 0: 对照基线
- 前后端同时输出 `legacy-v1` 与 `v2-research-hybrid` 的主质结果（不对用户展示差异）。
- 记录：
  - 主质一致率；
  - 次质分布；
  - 置信度分布；
  - 用户手动改动率。

### Phase 1: 建议模式
- UI 显示研究版主质 + 次质建议，但默认允许用户二次确认。
- 当 `confidence < 0.6` 时，显示“建议复测”提示。

### Phase 2: 半自动模式
- 对 `confidence >= 0.75` 的样本直接采用研究版主质；
- `confidence < 0.75` 继续保留手动确认。

### Phase 3: 全量切换
- 推荐链路默认使用研究版主质/次质；
- legacy 保留 1 个版本周期作为回归基线。

## 监控指标
- 画像确认率（用户未改动直接保存比例）
- 复测稳定性（同用户 14 天内一致率）
- 推荐命中率（用户点击/收藏/复购等业务指标）
- 算法漂移（各体质分布与历史均值偏差）

## 回滚策略
- 服务端支持通过 `questionVersion` 强制降级到 `legacy-v1`；
- 若出现异常（确认率骤降、投诉上升），立即回滚版本并冻结新题入库。

## 题库治理
- 新题进入正式库前必须通过：
  - 题项分布检查（避免极端偏态）；
  - 题项-总分相关下限；
  - 维度 alpha 下限；
  - 小流量线上表现观察。
