# 大模型调用与调试说明

本文档总结本项目（本地对话前端 + Node 代理）与上游大模型网关的对接方式、**curl 调试**、**接口参数**及**配置方法**，便于排查「网络 / DNS / 鉴权 / 模型 ID」等问题。

---

## 1. 架构说明

| 层级 | 说明 |
|------|------|
| 浏览器 | 访问 `http://<本机或局域网IP>:11011`，页面将「会话消息 + 模型连接信息」以 JSON 发给**同源**的 `/api/chat`。 |
| Node（`server.js`） | 监听 `0.0.0.0:11011`（可局域网访问），将请求 **转发** 到上游 OpenAI 兼容接口 `.../v1/chat/completions`，避免浏览器直连跨域（CORS）问题。 |
| 上游网关 | 示例地址：`http://ds.local.ai:30080/v1/chat/completions`，需内网 DNS 或 **hosts** 将 `ds.local.ai` 解析到实际 IP。 |

---

## 2. 内网域名（hosts）配置

若本机无法解析 `ds.local.ai`，需在 **Windows** 的 hosts 文件中增加映射（路径：`C:\Windows\System32\drivers\etc\hosts`，需管理员权限）：

```text
192.168.24.34 ds.local.ai
192.168.24.34 sre-nest.local.ai
192.168.24.34 vm.local.ai
192.168.24.34 harbor.local.ai
192.168.24.34 kibana.local.ai
192.168.24.34 n9e.local.ai
```

修改后建议执行（CMD 管理员）：

```bat
ipconfig /flushdns
```

## 3. 直连上游：curl 详细说明

### 3.1 接口信息

| 项 | 值 |
|----|-----|
| 方法 | `POST` |
| URL | `http://ds.local.ai:30080/v1/chat/completions`（可按实际网关修改主机与端口） |
| 鉴权 | HTTP Header：`Authorization: Bearer <App Key>` |
| 内容类型 | `Content-Type: application/json` |

### 3.2 请求体（JSON）主要字段

与 OpenAI Chat Completions 兼容，本项目实际发送字段如下。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model` | string | 是 | 平台分配的**模型 ID**，例如 `MDQWEN330BBF4AAE62D7EA`。 |
| `messages` | array | 是 | 对话消息列表；每项含 `role` 与 `content`。 |

**`messages[]` 单条结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `role` | string | 一般为 `user`、`assistant`、`system`（以网关文档为准）。 |
| `content` | string | 文本内容。 |

### 3.3 已提供的两套模型示例（来自业务配置）

| 模型名称 | 模型 ID（`model`） | App Key（Bearer） |
|----------|-------------------|-------------------|
| Qwen3-30B-A3B-Instruct-2507 | `MDQWEN330BBF4AAE62D7EA` | `de00e26309e14847964d42c871c8ffca` |
| Qwen3-VL-32B-Instruct | `MDQWEN3VL38EB081E75028` | `13f85efc3d9c43d5bd0351621c7ca895` |

> 生产环境请勿把密钥提交到公共仓库；文档中的 Key 仅作联调示例。

### 3.4 curl 示例（推荐：JSON 存文件，避免 PowerShell 转义问题）

**步骤 1**：创建 `body.json`（UTF-8 无 BOM 为佳）：

```json
{
  "model": "MDQWEN330BBF4AAE62D7EA",
  "messages": [
    { "role": "user", "content": "你是谁" }
  ]
}
```

**步骤 2**：在 **CMD** 中执行（将 `body.json` 换成你的实际路径）：

```bat
curl.exe -v --connect-timeout 15 --max-time 120 ^
  --location "http://ds.local.ai:30080/v1/chat/completions" ^
  --header "Authorization: Bearer de00e26309e14847964d42c871c8ffca" ^
  --header "Content-Type: application/json" ^
  --data-binary "@C:\path\to\body.json"
```

**curl 参数说明：**

| 参数 | 作用 |
|------|------|
| `-v` | 详细输出，便于看 DNS、TLS（若 https）、HTTP 状态码。 |
| `--connect-timeout 15` | 连接阶段超时（秒）。 |
| `--max-time 120` | 整次请求最大耗时（秒），大模型推理可适当加大。 |
| `--location` | 跟随 301/302 重定向。 |
| `--header "Authorization: Bearer ..."` | 网关要求的 Bearer Token。 |
| `--header "Content-Type: application/json"` | 声明 JSON 请求体。 |
| `--data-binary "@文件"` | 从文件读取原始请求体，避免命令行编码/转义错误。 |

**Linux / macOS** 可将换行续行符改为 `\`：

```bash
curl -v --connect-timeout 15 --max-time 120 \
  --location 'http://ds.local.ai:30080/v1/chat/completions' \
  --header 'Authorization: Bearer de00e26309e14847964d42c871c8ffca' \
  --header 'Content-Type: application/json' \
  --data-binary @./body.json
```

### 3.5 常见 curl / 网络结论

| 现象 | 可能原因 |
|------|----------|
| `Could not resolve host` | hosts 未配置、DNS 不可达或主机名写错。 |
| `Connection refused` | 目标 IP 上 30080 未监听或防火墙拦截。 |
| HTTP `401` / `403` | App Key 错误或无权访问该模型。 |
| HTTP `4xx` 且 JSON 含 `error` | 网关业务错误（模型 ID、参数格式等），以响应体为准。 |
| 返回 HTML 或非 JSON | 可能被反向代理/网关返回了错误页；代理层会返回 502 并附带 `raw` 片段（本项目）。 |

## 4. 本机代理接口：`POST /api/chat`

浏览器调用同源地址，例如：`http://127.0.0.1:11011/api/chat`。

### 4.1 请求头

| Header | 值 |
|--------|-----|
| `Content-Type` | `application/json` |

### 4.2 请求体（JSON）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `messages` | array | **是** | 与上游一致，非空数组。 |
| `apiUrl` | string | 否 | 完整上游 URL，须为 `http://` 或 `https://`；缺省使用服务端环境变量或内置默认。 |
| `apiKey` | string | 否 | Bearer Token；缺省使用服务端默认。 |
| `model` | string | 否 | 模型 ID；缺省使用服务端默认。 |

**示例：**

```json
{
  "messages": [
    { "role": "user", "content": "你好" }
  ],
  "apiUrl": "http://ds.local.ai:30080/v1/chat/completions",
  "apiKey": "de00e26309e14847964d42c871c8ffca",
  "model": "MDQWEN330BBF4AAE62D7EA"
}
```

### 4.3 代理行为摘要

1. 使用（请求体中的 `apiUrl` / `apiKey` / `model` 或）服务端默认值，对上游发起 `POST`。  
2. 上游响应体原样解析为 JSON；若成功则 **200** 返回给浏览器。  
3. 上游返回非 2xx：HTTP 状态码与 JSON 体尽量**透传**。  
4. 上游返回非 JSON：**502**，形如 `{ "error": "上游返回非 JSON", "raw": "..." }`。  
5. `fetch` 抛错（DNS、连接失败等）：**502**（网络类）或 **500**，形如 `{ "error": "...", "detail": "..." }`。

## 5. Node 服务端环境变量配置

在启动前设置（PowerShell 示例）：

| 变量 | 含义 | 默认（代码内） |
|------|------|----------------|
| `PORT` | 监听端口 | `11011` |
| `HOST` | 监听地址 | `0.0.0.0`（局域网可访问） |
| `LLM_URL` | 默认上游完整 URL | `http://ds.local.ai:30080/v1/chat/completions` |
| `LLM_API_KEY` | 默认 Bearer Key | 与 30B 示例一致 |
| `LLM_MODEL` | 默认模型 ID | `MDQWEN330BBF4AAE62D7EA` |

示例：

```powershell
$env:LLM_URL="http://ds.local.ai:30080/v1/chat/completions"
$env:LLM_API_KEY="你的Key"
$env:LLM_MODEL="MDQWEN3VL38EB081E75028"
npm run dev
```

启动后控制台会打印默认上游与模型；前端「系统配置」仍可覆盖单次请求使用的三项。

## 6. 前端「系统配置」

- 入口：页面侧栏 **「系统配置」**。  
- 存储：浏览器 `localStorage`，键名 **`local-ai-chat-config-v1`**，JSON 结构：

```json
{
  "apiUrl": "http://ds.local.ai:30080/v1/chat/completions",
  "apiKey": "……",
  "model": "……"
}
```

- 快速填入：界面提供 **Qwen3-30B** / **Qwen3-VL-32B** 预设按钮。  
- 说明：密钥保存在本机浏览器中，适合内网联调；对外发布请改为服务端保管。

## 7. 调试清单（建议顺序）

1. **hosts + flushdns**，确认 `ping ds.local.ai` 或 `curl` 能解析到预期 IP。  
2. **直连 curl**（第 3 节）成功后再测网页，区分「网关问题」与「代理/前端问题」。  
3. 看 **Node 控制台** `[api/chat] upstream fetch failed` 后的 `detail`（如 `ENOTFOUND`、`ECONNREFUSED`）。  
4. 浏览器 F12 → Network → **`/api/chat`** 查看状态码与响应 JSON。
