# medican 项目交付手册

本文面向**接收方运维 / 二次开发 / 验收方**，说明本仓库的交付范围、运行条件、启动方式、生产安全基线与验收要点。更细的日常开发约定见 [`README.md`](../../README.md) 与 [`AGENTS.md`](../../AGENTS.md)。

---

## 1. 交付范围与产品形态

| 项 | 说明 |
|----|------|
| 产品形态 | 浏览器 Web 应用（学生端 / 校园场景等）+ 管理端；后端提供 REST API |
| 前端工程 | `tcm-diet-frontend`（Vue 3 + Vite + Element Plus / Vant） |
| 后端工程 | `campus-diet-backend`（Spring Boot 2.7，**Java 11**） |
| 脚本与运维辅助 | 仓库根 `*.bat`、`scripts/*.ps1`、`docker-compose.mysql-dev.yml` 等 |
| 文档 | `README.md`、`docs/` 下清单与安全基线、本目录 [`大模型调用调试说明.md`](大模型调用调试说明.md) 等 |
| 离线交付压缩包（可选） | 由 `scripts/Build-MedicanDeliveryZip.ps1` 生成至 **`release/`** 目录，见下文 **§1.1** |

**不在交付物中的内容（需接收方自备或另行约定）**：生产数据库实例、域名与 TLS 证书、反向代理、密钥托管（KMS/Vault 等）、大模型服务端点与商业授权。

### 1.1 离线交付压缩包（数据库 SQL + 后端 jar + 前端 dist）

用于向接收方提供**不依赖本机 Git 检出**的可运行制品与建库脚本，默认打成一个 zip（**`release/medican-delivery-<时间戳>.zip`**，目录名形如 `medican-delivery-<时间戳>`）。

**生成命令**（在仓库根执行）：

```bat
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\Build-MedicanDeliveryZip.ps1
```

- 已构建过后端/前端、仅需重新打包时，可加 **`-SkipBackendBuild`** / **`-SkipFrontendBuild`**（需保证 `campus-diet-backend\target\*.jar` 与 `tcm-diet-frontend\dist` 已存在）。
- 加快打包且跳过单测：加 **`-SkipTests`**（仍执行 `package` 产出 jar）。
- **前端依赖安装**：默认若已存在 `tcm-diet-frontend\node_modules`，则**跳过 `npm ci`**，直接 **`npm run build`**（减轻 Windows 下文件占用导致的 EPERM）。需要干净安装时加 **`-NpmCi`**。若检测到缺少 `vite` 等关键依赖，脚本会尝试执行 **`npm install`** 做增量修复；构建前会将 **`node_modules\.bin`** 临时置于 **`PATH` 前缀**，以降低 Windows 下「找不到 vite」的概率。

**包内目录约定**：

| 路径 | 内容 |
|------|------|
| `database/` | `schema.sql`、`data.sql`、`patch-*.sql`；可选含 `mysql-dev-grant-tcm_app.sql`、`docker-compose.mysql-dev.yml` |
| `backend/` | Spring Boot 可执行 `*.jar`；`config/medican-datasource.override.example.yml`（MySQL 覆盖示例） |
| `frontend/dist/` | 前端静态资源（`npm run build` 产物） |
| `DELIVERY-BUNDLE.md` | 包内索引与运行提示（**英文**，避免脚本编码差异；中文验收仍以本手册为准） |
| `项目交付手册.md`、`README.md` | 与打包子时刻一致的文档副本 |

**与完整仓库交付的区别**：压缩包**不含**前端/后端完整源码、`node_modules`、CI 配置等；适合部署与验收介质归档。二次开发仍以 Git 仓库为准。

## 2. 技术栈与对外端口

| 组件 | 技术 | 默认监听 |
|------|------|------------|
| 前端开发服务 | Vite | `http://localhost:11999`（`strictPort: true`） |
| 后端 API | Spring Boot | `http://127.0.0.1:11888`；健康检查路径 **`/health`** |
| 前端 → 后端 | Vite 代理 | 浏览器请求 **`/api`** → 默认 **`http://127.0.0.1:11888`**（可用环境变量 **`VITE_PROXY_TARGET`** 覆盖） |

后端默认仅绑定 **`127.0.0.1`**。若需局域网或其它机器访问后端，需设置 **`SERVER_ADDRESS=0.0.0.0`**（同时应配合防火墙与 TLS/网关策略）。

## 3. 接收方环境前置条件

### 3.1 通用

- **Node.js**：建议 **18+**（用于前端 `npm install` / `build` / 测试）。
- **JDK**：**11**（须含 `javac`，勿用 JRE 8 跑 Maven）。
- **Maven**：全局安装 **或** 使用仓库内 **`campus-diet-backend/mvnw.cmd`**（首次需网络下载 Wrapper 指定版本到用户目录 `.m2/wrapper`）。
- **操作系统**：仓库脚本以 **Windows** 为主（`启动前端.bat`、`启动后端.bat`、PowerShell）；Linux/macOS 可参考 `README.md` 中的 Maven/测试命令自行等价替换。

### 3.2 可选 MySQL

- 默认后端 profile 为 **内存 H2**（`spring.profiles.default=h2`），**可不装 MySQL** 完成联调演示。
- 连接本机 MySQL 时：常见 **`127.0.0.1:3306`**、库名 **`tcm_diet`**；账号默认 **`tcm_app`**，口令勿使用仓库占位符上生产。授权与路径注意点见 [`README.md`](../../README.md) 中「MySQL」章节。

## 4. 标准交付验收：本地一键联调

下列步骤用于**功能与接口联调验收**（非生产部署）。

1. **启动后端**（仓库根）：双击 **`启动后端.bat`**。默认 H2，无需 MySQL。确认浏览器或命令行访问：`http://127.0.0.1:11888/health` 返回正常。
2. **启动前端**（仓库根）：双击 **`启动前端.bat`**（默认会探测后端 `/health`，可用环境变量 **`SKIP_BACKEND_CHECK=1`** 跳过）。访问：`http://localhost:11999`。
3. **种子账号**（开发/演示）：由 `SeedUsersRunner` 等逻辑保证存在；默认口令见 [`README.md`](../../README.md)「种子账号」段（如 `admin` / `admin123` 等）。**生产环境必须修改**，见第 6 节。

详细变量说明（`MYSQL_*`、`SPRING_DATASOURCE_URL`、`CAMPUS_JWT_SECRET`、`CAMPUS_CORS_ALLOWED_ORIGIN_PATTERNS` 等）见 [`README.md`](../../README.md)「环境变量速查」。

## 6. 生产环境交付要点（强制）

生产必须使用 **`prod` profile**，并满足 **`ProdSecurityBaselineValidator`** 与文档 **`docs/security-baseline-checklist.md`** 中的硬性要求。摘要如下（**不完整替代原文**）：

- **数据源**：用户名不得为空或 `root`；密码不得为空、不得为弱口令或与占位符相同。
- **种子用户口令**：不得保留开发默认值（如 `admin123`、`canteen123`、`demo123`、`123456`）。
- **`CAMPUS_JWT_SECRET`**：长度 ≥ 32，且不得包含开发用默认标识字符串。
- **`LLM_API_KEY`**：生产配置下须非空（若业务启用大模型能力）。
- **`spring.sql.init.mode`**：生产须为 **`never`**（与 `application-prod.yml` 一致）。
- **高风险种子开关**：生产须关闭 `seed-mock-recipes`、`seed-demo-interactions`、`seed-weekly-calendar` 等（见 `application-prod.yml` 与清单）。

**演练与证据**：按 [`security-baseline-checklist.md`](../security-baseline-checklist.md) 第 4 节执行；Windows 无 Bash 时可用仓库内 **`scripts/check-prod-security-baseline.ps1`**。

**密钥与覆盖配置**：JWT、数据库口令、LLM Key 等**不得**提交到 Git；优先使用部署环境的环境变量或受控密钥存储。数据源覆盖文件示例见 `campus-diet-backend/config/medican-datasource.override.example.yml`（仅 **`mysql` profile** 场景加载逻辑见 `application-mysql.yml`）。

## 7. 大模型（可选能力）

若验收范围包含 AI 能力：配置 **`LLM_URL`** 或 **`LLM_HTTP_HOST` / `LLM_HTTP_PORT`**、**`LLM_MODEL`**、**`LLM_API_KEY`**、**`LLM_TIMEOUT_MS`**。内网/本机联调说明见 **`大模型调用调试说明.md`**。前端开发模式默认可能启用 AI 相关 Vite mock；关闭方式见 [`README.md`](../../README.md)。

## 8. 常见问题（交付交接 FAQ）

| 现象 | 处理方向 |
|------|----------|
| `mvn` 找不到 | 使用 `campus-diet-backend\mvnw.cmd -s ..\maven-settings-d.xml …`，或配置全局 Maven；见 [`AGENTS.md`](../../AGENTS.md) |
| `java -version` 仍是 8 | 检查 **`JAVA_HOME`** 与用户 **`PATH`** 顺序，确保 JDK 11 在 Oracle `java8path` 之前 |
| 前端联调不到后端 | 确认后端已监听；检查 **`VITE_PROXY_TARGET`**；确认未误配仅内网可见的 `server.address` |
| 局域网无法访问后端 | 设置 **`SERVER_ADDRESS=0.0.0.0`** 并检查防火墙/安全组 |
| MySQL 授权/路径错误 | 从仓库根执行授权脚本；避免在错误工作目录下相对路径找不到 SQL；详见 [`README.md`](../../README.md) |
